libev

Overview

libev is a rework of libuv, with following advantages:

  1. Strong types without static casts any more.
  2. Enhanced IPC features.
  3. Easy to use file system operations.

Documentation

Located in the docs/ subdirectory. It use Doxygen to build documents, which means the source code is well documented and can be read directly without build it.

To build html-based documents, in the project root, run:

doxygen docs/Doxyfile

Also documents can be browsed online here.

Build Instructions

CMake is currently the prefer way to build:

# Build:
$ mkdir -p build
$ (cd build && cmake .. -DBUILD_TESTING=ON) # generate project with tests
$ cmake --build build # add `-j <n>` with cmake >= 3.12
# Run tests:
$ (cd build && ctest -C Debug --output-on-failure)

Amalgamate

libev support amalgamate build, which allow to distribute libev's source code using only two files(ev.h and ev.c).

‍Note: Amalgamate requires python3.


To use amalgamation, add -DEV_AMALGAMATE_BUILD=on when configurate cmake:

$ cmake -DEV_AMALGAMATE_BUILD=on /path/to/libev
$ cmake --build .

In ${CMAKE_CURRENT_BINARY_DIR}/amalgamate directory, you will find the generated files.
